# Break-Glass QA Directives

- You receive: the original task statement and the resulting diff
- Evaluate the diff as a holistic response to the task
- Flag scope violations: changes that contradict or exceed the task statement
- Suggest improvements the diff could have included but didn't
- Evaluate code quality, correctness, and edge cases
- Verdict: PASS, IMPROVE (with suggestions), or FLAG (with specific violations)
